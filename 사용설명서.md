# ⚾ 야구 스코어보드 사용설명서

이 문서는 **여러 사용자가 각자 야구 스코어보드를 운영**하기 위한 안내서입니다.
특히 **경기 결과가 저장되는 구글 시트를 사용자마다 따로 연결**하여 데이터가 섞이지 않게 하는 방법을 설명합니다.

---

## 1. 데이터가 분리되는 원리 (꼭 먼저 읽어주세요)

이 앱은 **앱을 배포한 구글 계정 소유의 스프레드시트**에 경기 결과를 저장합니다.

```
사용자 A가 배포한 앱  →  A의 구글 드라이브에 자동 생성된 시트에 기록
사용자 B가 배포한 앱  →  B의 구글 드라이브에 자동 생성된 시트에 기록
```

> 즉 **"앱을 각자 배포하면 결과 시트도 각자 따로 생긴다"** 가 핵심입니다.
> 하나의 배포본을 여러 명이 같이 쓰면 모든 결과가 **배포한 사람 한 명의 시트**에 모이므로,
> 데이터를 나누려면 **사용자마다 자신의 계정으로 앱을 배포**해야 합니다.

결과 시트의 이름은 `야구 스코어보드 경기기록` 이며, 첫 결과 저장 시 자동으로 생성됩니다.
한 사용자의 시트 안에서는 경기들이 `일시`·`팀명` 열로 구분되어 누적됩니다.

---

## 2. 내 계정으로 앱 배포하기

두 가지 방법이 있습니다. **개발 도구가 없어도 되는 [방법 A](#방법-a-웹-에디터에서-복사붙여넣기-비개발자-권장)** 를 권장합니다.

### 방법 A. 웹 에디터에서 복사·붙여넣기 (비개발자 권장)

준비물: 구글 계정 하나. (PC 권장)

1. **[script.google.com](https://script.google.com)** 접속 → 좌측 상단 **`+ 새 프로젝트`** 클릭.
2. 기본 생성된 `Code.gs` 파일의 내용을 모두 지우고, 이 저장소의 **`apps-script/Code.gs`** 내용을 통째로 붙여넣기.
3. 좌측 파일 목록의 **`+` → `HTML`** 선택 → 파일 이름을 정확히 **`Index`** 로 입력(확장자 `.html` 은 자동) →
   이 저장소의 **`apps-script/Index.html`** 내용을 통째로 붙여넣기.
   - ⚠️ `Index.html` 은 빌드 산출물입니다. 직접 수정하지 말고 **그대로** 붙여넣으세요.
4. 좌측 **⚙️ 프로젝트 설정** → **"`appsscript.json` 매니페스트 파일을 편집기에 표시"** 체크 →
   편집기에 나타난 `appsscript.json` 내용을 이 저장소의 **`apps-script/appsscript.json`** 내용으로 교체.
   (시간대 `Asia/Seoul`, 웹앱 접근 권한 등이 들어 있습니다.)
5. 우측 상단 **`배포` → `새 배포`** → 톱니바퀴(유형 선택)에서 **`웹 앱`** 선택.
   - **실행 계정:** `나(본인 이메일)`
   - **액세스 권한:** `모든 사용자`
   - **`배포`** 클릭.
6. 처음 배포하면 **권한 승인 화면**이 뜹니다 → 본인 구글 계정으로 로그인하고 **허용**.
   (이 단계를 건너뛰면 접속 시 `403` 오류가 납니다. → [4번 문제 해결](#4-자주-겪는-문제) 참고)
7. 배포 완료 후 표시되는 **웹 앱 URL(`.../exec`)** 이 본인 전용 스코어보드 주소입니다. 북마크해 두세요.

### 방법 B. clasp(명령줄)로 배포 (개발자용)

`Node.js` 와 터미널을 쓸 수 있는 경우.

```bash
# 1) 코드 받기 & 의존성 설치
git clone https://github.com/Kimjs99/baseball-scoreboard.git
cd baseball-scoreboard
npm install

# 2) 빌드 (src/app.jsx → apps-script/Index.html 재생성)
npm run build

# 3) 본인 구글 계정으로 로그인
clasp login

# 4) 본인 계정에 새 스크립트 프로젝트 생성 (clasp v3.x: --type standalone)
clasp create-script --type standalone --title "내 야구 스코어보드"
```

> ⚠️ `clasp create-script` 는 `appsscript.json` 을 기본값으로 덮어씁니다.
> 생성 직후 저장소의 `apps-script/appsscript.json`(웹앱 블록 + `Asia/Seoul`)을 **다시 적용**한 뒤 push 하세요.

```bash
# 5) 업로드 & 배포
clasp push
clasp create-deployment -d "최초 배포"   # 웹앱 /exec URL 생성
clasp open-web-app                        # 브라우저로 열기 → 권한 승인
```

## 3. 내 결과 시트 확인하기

> ⚠️ **저장은 반드시 배포된 웹앱 주소(`.../exec`)에서 하세요.**
> Apps Script 편집기의 **테스트/개발용 주소(`@HEAD`·`/dev`)는 로그인 필수**라 결과 전송이 거부되는데,
> 앱은 (no-cors 방식 특성상) 실패해도 **"전송 완료"로 표시**합니다. 기록이 시트에 안 남는다면 이 주소 문제일 가능성이 큽니다.
> [2번](#2-내-계정으로-앱-배포하기)에서 만든 `.../exec` 주소(북마크해 둔 주소)로 접속해 저장하세요.

1. 앱 화면에서 경기를 기록한 뒤 우측 상단 **`📤 결과 저장`** 클릭.
2. **`구글 시트로 전송`** 버튼 클릭. (URL 칸에는 본인 웹앱 주소가 이미 채워져 있습니다.)
3. **본인** 구글 드라이브([drive.google.com](https://drive.google.com))에서 **`야구 스코어보드 경기기록`** 파일을 열면
   `경기요약` · `타자기록` 두 시트에 결과가 쌓여 있습니다.
   - `경기요약`: 경기당 두 줄(어웨이/홈), 이닝별 점수와 R·H.
   - `타자기록`: 선수 한 명당 한 줄, 타수·안타·타율.
4. 같은 시트에 여러 경기를 누적해도 `일시` 열로 구분됩니다. 다른 사용자의 데이터는 들어오지 않습니다.

> 💡 시트를 다른 사람과 **보기 공유**하고 싶다면, 드라이브에서 해당 파일을 열어 직접 공유 설정을 하면 됩니다.
> (앱은 시트 공유 권한을 건드리지 않습니다.)

## 4. 앱 사용법 (기록 방법)

### 기본 액션
- **볼 / 스트라이크 / 파울** — 카운트 진행. 4볼은 자동 볼넷(밀어내기 포함), 3스트라이크는 삼진 아웃.
- **1·2·3루타 / 홈런** — 안타. 주자 진루·득점 자동 처리, 타수·안타 기록.
- **타격 아웃** — 일반 아웃(타수 기록).
- **되돌리기(Undo)** — 직전 동작을 한 단계 취소(모든 동작 취소 가능).
- **게임 리셋** — 전체 초기화(확인 후 실행).

### 특수 상황 (정확한 기록용)
| 버튼 | 타수(AB) | 안타 | 설명 |
|---|---|---|---|
| 볼넷 (BB) / 사구 (HBP) | 제외 | 제외 | 1루 진루권. 만루면 밀어내기 득점 |
| 야수선택 (FC) | **포함** | 제외 | 타자 출루, 다른 주자 아웃(아웃 +1) |
| 실책 출루 (ROE) | 포함 | 제외 | 출루 + 수비팀 실책(E) +1 |
| 희생플라이 (SF) | 득점 시 제외 | 제외 | 3루 주자 태그업 득점. 득점 없으면 일반 뜬공 |
| 희생번트 (SAC) | 제외 | 제외 | 주자 한 베이스씩 진루(3루 주자는 득점) |

### 주자 / 득점 수동 조정
도루·폭투·견제사·태그업처럼 버튼으로 딱 떨어지지 않는 상황은
**1루/2루/3루 주자 토글**로 주자를 직접 켜고 끄고, 득점이 나면 **`득점 +1`** 로 반영하세요.
이 동작들도 모두 **되돌리기(Undo)** 로 취소됩니다.

### 팀명·라인업 입력
우측 상단 **`⚙️ 팀 설정`** → 팀명과 선수 명단을 직접 입력하거나, 구글 시트(CSV) 링크로 불러올 수 있습니다.
- **타자 수(타순 인원)** 를 자유롭게 조정할 수 있습니다(1~20명). 입력한 인원만큼 타순이 순환하고 라인업·리더보드에 표시됩니다.
- 명단 칸을 클릭하면 기본 이름이 지워지며 바로 입력됩니다(이미 입력한 이름은 클릭 시 전체 선택되어 한 번에 교체).

### 경기 설정 (이닝 · 아웃 · 타자 수)
**`⚙️ 팀 설정`** → `경기 규칙`에서 **정규 이닝 수**, **이닝당 아웃 카운트**, **타자 수**를 바꿀 수 있습니다(경기 중에도 가능). 아웃 카운트를 바꾸면 B/S/O 패널의 O 표시도 함께 바뀝니다.

### 경기 종료 / 새 경기
- **`🏁 경기 종료`** — 현재 경기를 마감(액션 패널 잠금)하고 결과 저장 창을 엽니다.
- **`＋ 새 경기` / `새 경기(초기화)`** — 이번 경기 기록을 **전체(누적)** 에 합산하고 새 경기를 시작합니다.

### 라인업 기록 · 리더보드
- 스코어보드 좌(어웨이)·우(홈)에 타자별 **타수·안타·타율·타점** 이 표시됩니다. `이번 경기 / 전체 누적` 토글로 전환합니다.
- 상단 **`🏆 리더보드`** 탭에서 양 팀 타자 순위를 **타율·안타·타점** 기준으로 볼 수 있습니다.

## 5. 자주 겪는 문제

- **접속 시 `403` / 권한 오류** — 배포한 본인이 권한 승인을 안 한 상태입니다.
  본인 계정으로 웹앱 URL을 한 번 열어 동의하거나, 에디터에서 `authorize` 함수를 1회 실행하세요. (배포 계정만 하면 됩니다.)
- **화면이 하얗게 비어 있음** — `Index.html` 을 직접 수정했거나 잘못 붙여넣은 경우입니다.
  반드시 빌드된 원본 `apps-script/Index.html` 을 그대로 사용하세요. 수정이 필요하면 `src/app.jsx` 를 고치고 `npm run build` 후 다시 붙여넣습니다.
- **변경했는데 반영이 안 됨** — 코드 수정은 **다시 배포(또는 `clasp push` 후 재배포)** 해야 적용됩니다.
  반영 후에도 안 보이면 브라우저를 **강력 새로고침**(Ctrl/Cmd+Shift+R) 하세요. Apps Script가 화면을 강하게 캐시합니다.
- **결과 전송 후 시트가 안 보임** — 본인 드라이브에서 `야구 스코어보드 경기기록` 으로 검색하세요.
  첫 전송 때 자동 생성되므로, 한 번도 전송하지 않았다면 아직 없습니다.
- **"전송 완료"가 떴는데 기록이 안 남음** — 거의 대부분 **개발/테스트용 주소(`@HEAD`·`/dev`)에서 저장**했기 때문입니다.
  이 주소는 로그인 필수라 익명 저장이 거부되지만, 전송 방식(no-cors) 특성상 실패해도 성공처럼 보입니다.
  반드시 **배포된 `.../exec` 주소**에서 저장하세요. (코드를 새로 고쳤다면 `clasp push` 후 **`clasp redeploy`** 까지 해야 공유 주소에 반영됩니다.)
